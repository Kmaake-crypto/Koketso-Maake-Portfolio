Project Case Studies

Tesla Landing Page Clone

HTML CSS Netlify GitHub

What was the problem I was solving?

The goal was to build a Tesla inspired Landing Page Clone. The main challenge was building a layout that keeps the text in the middle of the screen and prevents large background images from looking distorted or cut off on smaller screens.

What decisions did I make and why?

I decided to use a layout that stretches to full screen height (100vh) and centered everything using CSS Flexbox. This makes the layout look neat, minimalist, and premium, exactly like the official brand style.

What tools did I use and what did I learn?

I used VS Code, Live Server,Github pages and Netlify to host the live site. I learned how to use special CSS background rules to make sure photos automatically scale nicely on both mobile phones and laptops.

What would I do differently? What am I proud of?

Next time, I will use CSS Grid for the footer links to keep the code shorter. I am super proud that the layout transitions perfectly on mobile viewports without breaking apart.

Netflix Landing Page Project

HTML CSS GitHub Netlify

What was the problem I was solving?

The goal was to build a close match of the Netflix landing page layout. The trickiest part was figuring out how to build an interactive accordion-style dropdown FAQ question block without relying on complex scripts that would over complicate the project and also uploading the trailer video using vs code without it breaking and making the line under the picture in the hero section curve.

What decisions did I make and why?

I chose a bold red and solid black color palette to capture the true identity of the platform. Instead of writing complex JavaScript code since well I have not understood the programming language as yet, I decided to use HTML dropdown tags to create the question boxes cleanly and keep the file lightweight.

What tools did I use and what did I learn?

I used HTML to map out the text, CSS to handle the immersive dark brand colors, and GitHub to track my progress safely and also Netlify to upload my landing page. I learned how to strip away native browser design defaults so that interactive components match a graphic mockup exactly.

What would I do differently? What am I proud of?

I would organize my layout styles using global CSS variables from day one to speed up my styling workflow. I am incredibly proud of achieving a high-fidelity visual layout that functions reliably using simple code.

YouTube Clone Project

HTML CSS JavaScript Netlify

What was the problem I was solving?

The goal was to build a responsive YouTube UI clone that mirrors the platform's layout and navigation structure. The challenge was creating a flexible grid system for video cards that adapts seamlessly across mobile, tablet, and desktop viewports while maintaining visual consistency with YouTube's iconic design.

What decisions did I make and why?

I chose to use CSS Grid for the video card layout instead of Flexbox because it provides better control over responsive breakpoints and automatic card wrapping. I implemented a dark theme with subtle hover effects to match YouTube's modern interface and keep user focus on video content.

What tools did I use and what did I learn?

I built this with HTML5 semantic markup, CSS Grid for layout, and JavaScript for interactive features like search filtering and sidebar navigation. I learned how to optimize DOM updates and manage state changes efficiently, particularly when handling user interactions with video lists.

What would I do differently? What am I proud of?

Next time, I would implement local state management from the start to handle complex UI interactions more elegantly. I'm proud of achieving a pixel-perfect replica of YouTube's responsive design and creating smooth transitions between different screen sizes.

Twitter Landing Page Project

HTML CSS JavaScript Netlify

What was the problem I was solving?

The objective was to recreate Twitter's landing page with a clean, modern design that captures the platform's minimalist aesthetic. The main challenge was building a responsive layout that maintains visual hierarchy and usability across all device sizes while staying true to Twitter's brand identity.

What decisions did I make and why?

I opted for a clean two-column layout with a left navigation sidebar and main content area on the right. I chose a light-on-dark color scheme with accent colors to highlight interactive elements, creating a premium feel that aligns with Twitter's professional brand positioning.

What tools did I use and what did I learn?

I used HTML for semantic structure, CSS Flexbox for flexible layout management, and JavaScript for interactive navigation and button behaviors. I learned how to create responsive sidebars that collapse on mobile and implement smooth state transitions for navigation elements.

What would I do differently? What am I proud of?

I would integrate more interactive features like real-time feed updates using JavaScript event listeners. I'm incredibly proud of recreating Twitter's signature aesthetic accurately and building a fully responsive interface that feels native and professional.

UI Screenshot Layout Project

HTML CSS Netlify

What was the problem I was solving?

The assignment was to take a raw design screenshot layout and translate it into clean, live front-end code. It required absolute, tight precision with element margins, custom grid positioning, and border rules to match the original design pixel-perfectly.

What decisions did I make and why?

I chose a modern "glassmorphism" visual style using transparent color layers (rgba) and backdrop blur effects. This created a clear physical layer of depth over my dark background graphics without reducing text readability, elevating the overall visual sophistication.

What tools did I use and what did I learn?

I relied on structured HTML markup and advanced CSS properties like custom shadows, filters, and precision box-sizing. I learned exactly how stacking layers work using the z-index property to float panels elegantly over underlying imagery and background textures.

What would I do differently? What am I proud of?

I would plan out smooth hover state animations for all interaction points right at the initial wireframe step. I am most proud of turning a high-end graphic design concept seamlessly into clean, stable production code with flawless visual fidelity.
